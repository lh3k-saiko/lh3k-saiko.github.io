## 事前工作

以下是本文会用到的一些定义及其记号。注意，这些定义并非十分严谨，严谨定义请参考学术权威资料。

- **若干**：在不产生歧义的情况下，指任意多个，**包括零个**。
- **字符串**：由若干个*字符*拼接而成的*序列*，*通常*使用（习惯为大写）拉丁字母表示。如字符串 $S=\texttt{ababz}$。
  - $|S|$ 表示字符串 $S$ 的串长。在上例中，$|S|=5$。
  - 字符串的下标从 $0$ 开始，最后一个字符的下标为 $|S|-1$。
  - $S[i]\ (0\le i\le|S|)$ 表示字符串 $S$ 的第 $i$ 个字符。在上例中，$S[0]=\texttt{a}$，$S[1]=\texttt{b}$，$S[2]=\texttt{a}$，$S[3]=\texttt{b}$，$S[4]=\texttt{z}$。特别的，$S[|S|]=\\#$，其中 $\\#$ 是一个特殊的字符，它的*字典序*小于任何其他字符，不计入串长。
- **空串**：长度为 $0$ 的字符串，记为 $\epsilon$。
- **前缀**：将一个字符串的结尾若干字符剔除后得到新串称为其前缀。如上例中 $\epsilon$，$\texttt{ab}$，$\texttt{abab}$ 和 $\texttt{ababz}$ 都是 $S$ 的前缀。
- **后缀**：将一个字符串的开头若干字符剔除后得到新串称为其后缀。如上例中 $\epsilon$，$\texttt{bz}$，$\texttt{abz}$ 和 $\texttt{ababz}$ 都是 $S$ 的前缀。
- **子串**：一个字符串的后缀的前缀为其子串。
  - $S[l..r]\ (0\le l\le r\le|S|)$ 表示将 $S$ 中第 $l$ 个字符到第 $r$ 个字符抽出组成的子串，如上例中 $S[1..3]=\texttt{bab}$。

一些字符串的其他定义，如*字典序*，将会于本文的其他章节或者其他文章中给出。

## 字符串相等

两个字符串 $S$ 和 $T$ 相等当且仅当 $|S|=|T|$ 且 $\forall i\isin\lbrace0,1,\dots,|S|-1\rbrace,\ S[i]=T[i]$。

显然，字符串相等具有传递性，即若 $S=T$，$T=U$，则 $S=U$。

判断 $S=T$ 是否成立的时间复杂度是 $O(\min(|S|,|T|))$。

## 字符串拼接

将字符串 $S$ 和 $T$ 首尾相接而成的字符串记为 $S+T$。如 $S=\texttt{abaca}$，$T=\texttt{cbabdf}$，则有 $S+T=\texttt{abacacbabdf}$。

特别的，记 $S^{k}=\underbrace{S+S+\dots+S}_{k \text{ 个 } S}$，$S^{\infty}$ 是一个无限长的，由无数个 $S$ 拼接而成的字符串。

单次字符串拼接的复杂度依赖于具体实现和场景，但一般不会超过 $O(|S|+|T|)$。

## 字典序

若一个字符串中的字符是可比较大小的（如按照字符的 ASCII 编码比较），则我们可以定义两个字符串之间的*字典序*大小关系。具体来说，我们认为两个字符串 $S<T$ 当且仅当

1. $S[0] < T[0]$
2. $S[0] = T[0]$ 且 $S[1..|S|-1]<T[1..|T-1|]$

中**任意一个**成立。

例如，按照字符的 ASCII 编码，$\texttt{a}<\texttt{abaa}<\texttt{aab}<\texttt{aabac}<\texttt{aaa}<\texttt{b}$。

## 字符串哈希

任意一个参数为任意字符串的函数 $H(S)$ 都可称为字符串的哈希函数。

由函数（映射）的性质，若 $H(S_0)\ne H(S_1)$，则 $S_0\ne S_1$。但是若 $H(S_0)=H(S_1)$ 却不一定有 $S_0=S_1$。我们将事件 $S_0\ne S_1\ |\ H(S_0)=H(S_1)$ 称为*冲突*。

好的字符串哈希函数应当具有以下性质：

1. 冲突发生的概率 $P\lbrace S_0=S_1\ |\ H(S_0)\ne H(S_1)\rbrace$ 应尽可能小；
2. 用于计算字符串哈希的函数的算法应当尽可能高效。

下面是一种常见的字符字符串哈希的实现方式。

```cpp
uint64_t myhash(const std::string &s) {
    constexpr uint64_t P = 9361973132609ull;
    constexpr uint64_t B = 233ull;
    uint64_t res = 0;
    for(char c: s) res = (res * B + c) % P;
    return res;
}
```

## 单模式串匹配

> 给定两个字符串 $S$ 和 $T$，$|T|\le|S|$，求 $\lbrace {i\ |\ S[\ i\ ..\ i+|T|-1\ ]=T}\rbrace$。

使用朴素算法可以在 $O(|S|\times|T|)$ 的时间内解决这一问题。使用字符串哈希可以在 $O(|S|+|T|)$ 的时间内解决这一问题，但是有出错的可能。使用 KMP 或者 Z 函数则可以在 $O(|S|+|T|)$ 的时间内解决这一问题。

不难知解决这一问题的时间复杂度下限为 $O(|S|+|T|)$，但是若 $S$ 和 $T$ 没有高度的循环性，则一些算法可以做到平均 $O\left(\frac{|S|}{|T|}\right)$。

## 字典树（Trie）

字典树是一颗*有根树*。树上的每一条边上都带有一个字符；每一个节点都代表一个字符串，具体而言

1. 根节点代表空串；
2. 设一个节点的父节点代表的字符串为 $S$，其与父节点的边上有字符 $c$，则该节点所代表的字符串为 $S+c$ 。

字典序具有如下性质：

1. 若一个字符串 $S$ 被字典树中的其中一个节点表示，则 $S$ 的所有前缀都能在字典树中找到表示其的节点，且这些节点是表示 $S$ 的节点的祖先；
2. 字典树中两个节点的*最近公共祖先*所表示的字符串是这两个节点所表示的字符串的*最长公共前缀*。

下面是一种通过动态插入字符串来实现构建字典树的代码实现。（仅作参考）

```cpp
struct Trie {
    std::map<char, Trie*> ch;
    void insert(const std::string &s) {
        Trie *cur = this;
        for(char c: s) {
            auto it = cur->ch.find(c);
            if(it == cur->ch.end())
                cur = cur->ch[c] = new Trie;
            else
                cur = it->second;
        }
        return ;
    }
};
```

## 多模式串匹配

> 给定字符串 $S$ 和 $T_1,T_2,\dots,T_n$，$\max_{i=1}^n\lbrace|T_i|\rbrace\le|S|$，求 $\lbrace {(i,j)\ |\ S[\ i\ ..\ i+|T_j|-1\ ]=T_j}\rbrace$。

这一类问题通常有两种方法给出，一种是先给出*模式串* $T_1,T_2,\dots,T_n$，再给出*文本串* $S$，另一种则是反过来。先给出模式串的问题可以使用 AC 自动机在线求解，先给出文本串的问题则需要使用诸如后缀数组、后缀自动机之类的算法。两类问题可以通过将询问离线相互转化。
