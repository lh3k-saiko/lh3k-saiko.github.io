# Instruction

## 操作数和寻址方式

x86 指令集的格式均为：操作码 + 0~3 个操作数。操作数共有四类：

- 寄存器操作数：操作数在 CPU 的通用寄存器或段寄存器中；
- 立即数操作数：操作数在指令中直接给出；
- 存储器操作数：操作数在存储器的存储单元中；
- I/O 操作数：操作数在输入/输出端口的寄存器中。

由此可以得到四种最基础的寻址方式：

- 寄存器寻址：直接从寄存器中读/写数据，如 `MOV AX, BX`（将 `BX` 的内容移动到 `AX`） 中的 `AX` 和 `BX`。
- 立即数寻址：直接从指令中读取的数据，如 `MOV AX, 1024`（将 `1024` 写入 `AX`）中的 `1024`。
- 储存器寻址：从储存器中寻址，这一部分较为繁琐，请看下一节。
- I/O 寻址：从 I/O 接口读入数据，通常和一些微机接口的指令有关。

### 储存器寻址

> 吐槽一句，masm 汇编语言中关于寻址的语法相当混乱，一个变量既可以当指针又可以当引用，方括号的作用更是模糊。建议有些 corner cases 直接就不管了。

采用存储器寻址的指令操作数可以由以下三个分量组合而成

- 位移量：一个常数或变量名+/-一个常数得到的偏移量
- 基址：由通用寄存器提供内容的地址
- 变址：由除 ESP 外寄存器提供内容的地址（变址寄存器还可以乘以比例因子 2、4、8）

最终的地址则是这三个位移量相加得到。

此外还需要注意的是有些指令并没有显示包含寻址，但是实际却约定俗成使用了一些寄存器，如串操作指令使用 `ESI` 作为源，`EDI` 作为目的。`MUL BL` 的另一个乘数为 `AL`，结果储存于 `AX` 中。

#### 直接寻址

只有偏移量的寻址方式称为直接寻址。如 `MOV AX, DATA`，`MOV AX, [DATA]`，`MOV AX, DATA+2`，`MOV AX, [DATA+2]`。若 `DATA=10086H` 数据段所指向的数据为 `12H 34H 56H 78H`，则执行完上述四条语句后 `AX` 的值分别为 `3412H`、`3412H`、`7856H` 和 `7856H`。

如果想要将 `DATA` 的偏移量存入 `AX` 中，可以写成 `MOV EAX, OFFSET DATA` 或者 `LEA EAX, DATA`。

需要注意的是，`MOV AX, 1145H` 和 `MOV AX, [1145H]` 都会被视为立即数寻址，其意义是将 `1145H` 写入寄存器 `AX` 中。如果确实需要读取偏移为 `00001145H` 处的数据，请先将这一数据写入一个寄存器中。

#### 寄存器间接寻址

只使用基址或者变址的寻址方式称为寄存器间接寻址。如 `MOV AX, [EBX]`、`MOV AX, [ECX*2]`、`MOV [ESI], AX`、`MOV [ESP], BX` 均是寄存器间接寻址。

需要注意的是，寄存器间接寻址必须打中括号，且当使用 `ESP` 寄存器时，不能乘以比例因子。

#### 寄存器相对寻址

在寄存器间接寻址方式上增加了一个偏移量的寻址方式称为寄存器相对寻址。如 `MOV AX, [EBX]+2`、`MOV CX, [EDX]4`、`MOV ESI, [EDI+6]`、`MOV EDI，[ESP+20]`、`MOV TABLE[EAX], 1145H`。

需要注意的 `MOV AX, [EBX]10` 时错误的语法，因为在省略乘号时偏移量不能直接写在中括号右边。

#### 基址变址寻址

同时拥有基址、变址和偏移量的寻址方式。如 `MOV AX, TABLE[EBX][ECX]`。

引用基址变址寻址的目的是为了方便处理二维数组？（不愧是 CISC 架构）

#### 串操作寻址

和串操作指令相关，具体请看之后的操作相关指令。

## 指令集

### 传送类指令（`MOV MOVZX MOVSX`）

这三类指令都用于将源操作数的内容传至目的操作数中。`MOV` 指令中源操作数和目的操作数的宽度必须相同，而 `MOVZX` 和 `MOVSX` 则要求目的操作数的宽度必须大于源操作数。`MOVZX` 进行无符号零拓展，而 `MOVSX` 进行有符号补妈拓展。 

需要注意的是，`CS/ECS` 不能作为 `MOV` 指令的目的操作数。储存单元和段寄存器个自内部之间不可直接传输数据（但是这俩者之间可以直接 `MOV`），需要使用通用寄存器中转。

**对标志寄存器的影响**：都没有

### 堆栈操作（`PUSH PUSHF PUSHFD PUSHA PUSHAD POP POPF POPFD POPA POPAD`）

在 16 位（实模式）下所有以上操作均只对 16 位操作数有效，32 位模式下 `PUSH/POP` 只能操作 32 位数。（Note：这一点有待考证）

- `PUSH SRC`/`POP DEST`：将一个数据压入栈中。此外 `POP` 不能将数据弹出到 `CS` 中。
- `PUSHFD?`/`POPFD?`：将标志寄存器 `FLAGS` 压入/弹出栈中。后缀若有 `D`(Double) 则压入的是 32 位，否则位 16 位的。
- `PUSHAD?`：将照 `AX BX CX DX SP BP SI DI` 压入栈中，若后缀加有 `D` 则压入的是对应 32 位的（即 `E` 开头的）寄存器。
- `POPAD?`：按照 `DI SI BP SP DX CX BX AX` 弹出栈，若后缀加有 `D` 则弹出的是对应 32 位的（即 `E` 开头的）寄存器。

**对标志寄存器的影响**：除了 `POPFD?` 会整体改写标志寄存器外，其余均没有

#### 交换指令（`XCHG`）

`XCHG DEST, SRC`，用于交换两个等宽数据，可以是通用寄存器之间，也可以是通用寄存器和储存单元之间。但不能是储存单元和储存单元间，也不能使用段寄存器作为源或目的操作数。

**对标志寄存器的影响**：没有

### 标注传送指令（`LAHF` 和 `SAHF`）

- `LAHF`：将标志寄存器的低八位（其中包括 `SF ZF AF PF CF`）传入 `AH`。
- `SAHF`：将 `AH` 送入标志寄存器的低八位中。

**对标志寄存器的影响**：LAHF 不对标志寄存器产生影响，`SAHF` 会改变其低八位。


### 加减指令（`ADD ADC INC SUB SBB DEC NEG CMP`）

- `ADD ADC SUB SBB`：进行加/减法，其中 `ADC/SBB` 在执行时还会带上 `CF` 的值。
- `INC DEC`：将操作数加一/减一。
- `NEG`：将操作数取反。
- `CMP`：计算两个数的差值，但是只改变标志寄存器，结果不写入目的寄存器。

**对标志寄存器的影响**：

- `ADD ADC SUB SBB NEG CMP` 改变 `CF PF AF ZF SF OF` 位
- `INC DEC` 改变 `PF AF ZF SF OF`（注意不影响 `CF` 位）
- 值得注意的是，`NEG` 指令相当于 `DEST:=0-DEST`。当 `DEST` 的值当前位宽做能表示最小的负数时（如 8 位下的 `-128` 和 32 位下的 `80000000H`），由于没有对应的正数，这会使得目的操作数不变，但是 `OF` 置为 1。（其余情况 `OF` 置 0）此外当 `DEST` 为 0 是 `CF` 置 0，其余情况下 `CF` 置 1。

### 乘除指令和（`MUL IMUL DIV IDIV`）

操作码前带有 `I` 的为有符号数操作，没有的则为无符号数。

| 指令 | 位数 | 格式 | 操作 | 备注 |
|:---:|:----:|:----|:----|:-----|
| `I?MUL` | 8 | `I?MUL OPRD` | `AX := AL * OPRD` | 亦 `AH:AL := AL * OPRD` | 
| `I?MUL` | 16 | `I?MUL OPRD` | `DX:AX := AX * OPRD` | |
| `I?MUL` | 32 | `I?MUL OPRD` | `EDX:EAX := EAX * OPRD` |  |
| `I?DIV` | 8 | `I?DIV OPRD` | `AL = AX / OPRD`<br>`AH = AX % OPRD ` | 在除数为 0 或商溢出时会产生 0 中断 | 
| `I?DIV` | 16 | `I?DIV OPRD` | `AX := DX:AX / OPRD`<br>`DX =: DX:AX % OPRD ` | 在除数为 0 或商溢出时会产生 0 中断 | 
| `I?DIV` | 32 | `I?DIV OPRD` | `EAX := EDX:EAX / OPRD`<br>`EDX := EDX:EAX % OPRD ` | 在除数为 0 或商溢出时会产生 0 中断 |

此外 `IMUL` 在 32 位保护模式下还有双操作数和三操作数的形式，和高级语言比较类似，在此不再展示。

**对标志寄存器的影响**：

- `MUL` 指令当结果的高半部分非零时置 `CF=OF=1`，否则置 `CF=OF=0`。对于 `SF AF ZF PF` 的影响未定义。
- 单操作数 `IMUL` 指令当结果的高半部分不是低半部分的符号拓展时置 `CF=OF=1`，否则置 `CF=OF=0`。对于 `SF AF ZF PF` 的影响未定义。
- 双操作数和三操作数 `IMUL` 指令当结果的仅保留低半部分时仍然正确置 `CF=OF=0`，否则置 `CF=OF=1`。对于 `SF AF ZF PF` 的影响未定义。
- `DIV` 和 `IDIV` 指令对于 `CF OF SF ZF AF PF` 的影响未定义。

### 字位拓展指令（`CBW CWD CDQ`）

通常用于做除法指令的位拓展。

- `CBW`：将 `AL` 符号拓展到 `AH`
- `CWD`：将 `AX` 符号拓展到 `DX`
- `CDQ`：将 `EAX` 符号拓展到 `EDX`

### BCD 码调整指令（`AAA DAA AAS DAS AAM AAD`）

BCD 码分为两种，一种是组合型，一个字节表示两个十进制数；另一种是非组合型的，一个字节仅第四位用于表示十进制数，高四位则全置 0 或 1。

`AAA AAS AAM AAD` 用于矫正非组合型 BCD 码加减乘除的运算结果，而 `DAA DAS` 用于矫正组合型 BCD 码的运算结果。

### 逻辑运算指令（`AND TEST OR XOR NOT`）

- `AND TEST`：做按位与运算，区别在于 `TEST` 不将结果写回目的操作数。
- `OR`：做按位或运算。
- `XOR`：做按位异或运算。
- `NOT`：做按位取反运算。

**对标志寄存器的影响**：根据运算结果设置 `PF ZF SF`，`CF` 和 `OF` 永置 0，`AF` 未定义。

#### 位移指令（`SAL SAR SHL SHR SHLD SHRD ROL ROR RCL RCR`）

上述指令在非 32 位保护模式下使用时若位移次数不为 1 则需要使用 `CL` 作为位移次数寄存器。

带 `D` 后缀的为三操作数指令，相当于将前两个数看作一个整体进行位移。这些指令只能在 32 位保护模式下使用。

| 指令名 | 方向 | 位移/循环方式 | 填充方式 | 对 `CF` 的影响 |
|:-----:|:----:|:-----------:|:------:|:-------------|
| `SAL` | 左 | 算数左移 | 低位 0 填充 | 将左移出的最后一位至 `CF` |
| `SAR` | 右 | 算数右移 | 高位符号填充 | 将右移出的最后一位至 `CF` |
| `SHL SHLD` | 左 | 逻辑左移 | 低位 0 填充 | 将左移出的最后一位至 `CF` |
| `SHR SHRD` | 右 | 逻辑右移 | 高位 0 填充 | 将右移出的最后一位至 `CF` |
| `ROL` | 左 | 循环左移 | （无） | 将左移出的最后一位至 `CF` |
| `ROR` | 右 | 循环右移 | （无） | 将右移出的最后一位至 `CF` |
| `RCL` | 左 | 连带 `CF` 的循环左移 | （无） | 将 `CF` 作为最高位一起左移 |
| `RCR` | 右 | 连带 `CF` 的循环右移 | （无） | 将 `CF` 作为最高位一起右移 |

**对标志寄存器的影响**：

无论什么样的指令，若循环位移为 0，则对标志无影响。

所有以上指令会根据运算结果设置 `PF ZF SF`，而对 `AF` 的影响均未定义。而对于 `CF` 的影响如上表所述。

对于 `OF`，若只位移了一位，则 `OF` 的结果为符号位是否发生改变；否则 `OF` 为未定义。

### 位测试指令（`BT BTC BTS BTR BSF BSR`）

这六个指令均为双操作数指令。前四者用于将一个数二进制下的某一位送入 `CL` 中。对于 `BTC BTS BTR`，在送入 `CL` 后还会对这一位分别进行取反、置 1 和置 0 操作。`BSF BSR` 相当于 countr_zero 和 countl_zero。

**对标志寄存器的影响**：

`BT BTC BTS BTR` 会将结果写入 `CL` 中，不会影响 `ZF`。对于 `OF SF AF PF` 的影响未定义。

`BSF BSR` 会根据源操作数是否为零设置 `ZF`（`ZF:=!SRC`）。对于 `CF OF SF AF PF` 的影响未定义。

### 串操作指令

未有特殊说明，所有接下来的指令都有 8/16/32 位长度的版本，分别对应其加上后缀 `B W D` 后的指令。如 `LODS` 的 32 位版本为 `LODSD`。

`LODS` 执行同时还会“移动” `ESI`，`STOS` 则会“移动” `EDI`。其他指令则会移动两者。移动的方向和长度由标志寄存器中的 `DI` 和操作数的大小决定，`DI=0` 向前，`DI=1` 向后。

#### `LODS`

将 `[ESI]` 写入 `EAX`（或者其低位） 并移动 `ESI`。

**对标志寄存器的影响**：无。

#### `STOS`

将 `EAX`（或者其低位）的内容写入 `[EDI]` 并移动 `EDI`。

**对标志寄存器的影响**：无。

#### `MOVS`

将 `[ESI]` 写入 `[EDI]` 并同时移动两个指针。

**对标志寄存器的影响**：无。

#### `CMPS`

比较 `[ESI]` 和 `[EDI]` 并同时移动两个指针。

**对标志寄存器的影响**：和 `CMP [ESI] [EDI]` 相同。

#### `SCAS`

比较 `EAX`（或者其低位）和 `[EDI]` 并移动 `EDI`。

**对标志寄存器的影响**：和 `CMP (E?AX|AL) [EDI]` 相同。

#### `REP(E|Z|NE|NZ)?`

这一指令后面紧跟上述某一个串指令，并重复这一指令 `ECX` 次。（类似于 `while(ECX--) s_inst();`）

若有后缀 `E|Z`，则会在 `ZF!=0` 时退出循环。（类似于 `while(ECX-- && ZF == 0) s_inst();`）

若有后缀 `NE|NZ`，则会在 `ZF==0` 时退出循环。（类似于 `while(ECX-- && ZF != 0) s_inst();`）

### 程序控制指令

#### 无条件转移 `JMP`

无条件转移分为三类：短转移、近转移和远转移。短转移范围在 $-128$ 到 $127$ 之间；近转移在 $-2^{31}$ 到 $2^{31}-1$ 之间，但仍在一个代码段内；远转移则会跳到另外的代码段，一般用于多程序的链接。近转移的 `JMP` 指令为 2 字节长，而远转移则为 5 字节。此外 `JMP` 指令也有直接寻址和间接寻址两种方式。

需要注意的是 `JMP` 指令的操作数是有正负的，计算时若给出的是立即数则需先判断其正负。

**对标志寄存器的影响**：没有。（PS：仅针对没有 task switch 的情况）

#### 条件跳转指令 `JXX`

条件跳转指令均为短转移/近转移。

条件跳转指令相当多。常见缩写词可以参考下面的表格。

| 缩写词 | 英文全称 | 含义 |
|:-----:|:-------:|:------:|
| `A` | Above | 无符号数大于 |
| `B` | Below | 无符号数小于 |
| `C` | Carry | `CF=1` |
| `E` | Equal | 相等 |
| `G` | Greater | 有符号数大于 |
| `L` | Less | 有符号数小于 |
| `N` | Not | 非 |
| `O` | Overflow | `OF=1` |
| `P` | Parity | `PF=1` |
| `PE` | Parity Even | `PF=1` |
| `PO` | Parity Odd | `PF=0` |
| `S` | Sign | `SF=1` |
| `Z` | Zero | `Z=1` |

完整表格可以[参考这里](https://www.felixcloutier.com/x86/jcc)。
 
**对标志寄存器的影响**：均没有。

#### 循环跳转 `LOOP(Z|E|NZ|NE)?`

`LOOP` 系列的语句都先会令 `ECX:=ECX-1`，然后判断 `ECX` 是否为 0。如果非 0 则继续执行，否则跳转。

带有 `Z|E` 后缀的发生跳转还需要 `ZF=1`，带有 `NZ|NE` 后缀的则还需要 `ZF=0`。

值得注意的是如果在进入循环前 `ECX` 就为 0，则会造成一个近乎死的循环（32 位下循环 `2^32` 次）。进入循环主体前可以使用 `JECXZ` 来判断 `ECX` 是否为 0。如果为 0 则直接跳过循环。

**对标志寄存器的影响**：均没有。

#### 过程调用和返回（`CALL RET`）

- `CALL`：将 `EIP` 压入栈并跳转到对应程序处。和 `JMP` 指令类似也有直接寻址和间接寻址两种跳转方式。
- `RET`：分为不带操作数和带一个**偶数**操作数的版本。两个版本的指令都会先从栈顶弹出四个字节的内容写入 `EIP`，带有操作数 `n` 的还会从栈顶额外弹出 `n` 个字节。

**对标志寄存器的影响**：除了 task switch 以外的情况均没有。

### 标志位控制指令

一共有七个，都与标志位有关：

- `CLC`：`CF:=0`
- `SLC`：`CF:=1`
- `CMC`：`CF:=!CF`
- `CLD`：`DF=0`
- `SLD`：`DF=1`
- `CLI`：`IF=0`
- `SLI`：`IF=1`

速记：`CL` 开头为置 0，`SL` 开头为置 1，`CMC` 是唯一一个清空的。

`CF DF IF` 是程序中为数不多的需要主动设置的标志位。

### CPU 控制指令

#### `NOP`

什么都不做。

FUN FACT：这个一个字节长的指令实际上进行的操作是 `XCHG E?AX E?AX`。

**对标志寄存器的影响**：无。

#### `HLT`

将当前处理器设置为挂起，直到收到复位信号或者外部中断。如有外部中断请求，则 CPU 会先执行中断程序，再执行 `HLT` 之后的程序。

这是一个相当危险的指令，执行 `HLT` 需要 0 级权限。

**对标志寄存器的影响**：无。

#### `WAIT`

> <span class="text-red-500">这似乎是一个很老旧的指令，请不要在真实的汇编场景中使用。</span>

让 CPU 处于等待状态，接收外部中断请求后执行中断任务。在中断任务完成后回到等待状态。

**对标志寄存器的影响**：无。

#### `ESC`

> <span class="text-red-500">这似乎是一个很老旧的指令，请不要在真实的汇编场景中使用。</span>

格式：`ESC OPRD, SRC`

功能（我猜的）：将 `SRC` 的内容送到数据总线上，并让其他设备完成一个由 `OPRD` 所指定的功能。`OPRD` 是一个 6 位的协议码，至于 `OPRD` 指定的是什么功能需要开发者实现制定好协议。

**对标志寄存器的影响**：无。

#### `LOCK`

`LOCK` 是一个指令前缀。加了 `LOCK` 前缀的指令将会独占总线直到执行完成。用于在多任务场景中实现原子操作。

**对标志寄存器的影响**：无。

### 输入输出指令

x86 系列中的端口都是独立编址的，并且使用专用的 `IN` 和 `OUT` 的指令来进行数据的读取。端口地址也有直接寻址和间接寻址两种方式。其中直接寻址只能使用 0~255 作为端口号，间接寻址只能使用 16 位寄存器 `DX` 的内容作为端口号。且 `IN` 写入和 `OUT` 输出的对象只能是 AL/AX/EAX 三者之一（这在 64 位中仍然适用）。

理论上最多有 65536 个端口号，但实际上远远用不到那么多。`IN` 和 `OUT` 的具体适用可以参考后面的微机接口部分。

**对标志寄存器的影响**：None。
