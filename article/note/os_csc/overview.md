# 概述

## 操作系统的目标、作用和模型

操作系统是一组控制和管理计算机硬件和软件资源，合理地对各类作业进行调度，以及方便用户使用的**程序的集合**。

> An operating system (OS) is system software that manages
> computer hardware and software resources and provides
> common services for computer programs.
> 
> <p class="text-right italic">——wikipedia</p>

操作系统的由三个目标：

- 便利性：使计算机更易于使用
- 有效性：允许以更有效的方式使用计算机系统资源
- 扩展能力：在不妨碍服务的前提下，有效地开发、测试和引入新的系统功能

对于用户而言，操作系统提供的服务有：

- 程序开发：提供一系列工具和服务（如编辑器和调试器）
- 程序运行：为用户处理调度问题，如加载到内存、初始化 I/O 设备等
- I/O 设备访问：提供统一接口，隐藏具体的 I/O 操作指令
- 文件访问控制：屏蔽存储介质细节
- 系统访问：提供接口，防止未授权访问行为
- 错误检测和响应：软、硬件错误
- 日志：收集资源的利用率信息、监控性能特性

而作为资源管理器，操作系统的则主要管理数据的移动、存储和管理。

需要注意的是，占用处理器的时间也是一种资源。而 OS 也是一个或一组程序。所以 OS 会转交控制权，且须依赖处理器才能恢复控制。

此外操作系统还要有一定的**易发展性**。即在硬件设备升级换代及一定程度的错误发生时仍然能够稳定地为用户提供服务。

## 操作系统地发展过程

主要为从串行处理（无操作系统）、简单批处理系统、多道批处理系统、分时系统和实时系统构成。

### 串行处理

即没有操作系统，用户按顺序通过控制台来访问计算机，一次执行一个程序，作业的载入和切换由人工完成。

### 简单批处理系统

- 早期操作系统，有监控程序来完成作业的自动续接，并且能够完成简单的保护、资源管理和中断。
- 有用户模式和特权模式，对资源访问提供一定程度的保护。
- 仍然一次只能运行一个程序，内存中一次也只能有一个任务。CPU 使用率很低。

### 多道批处理系统

- 内存中可以存放多个作业、多个作业并发执行
- 需要由硬件提供 I/O 中断和直接储存器访问（DMA）等功能
- 由作业调度程序来负责作业的调度
- 多道并行使得系统呈现多道性、调度性、**无序性**
- 此外多道并行系统没有交互能力

而多道作业并发处理概念的引入是的系统不可避免地需要解决对于资源（如内存、外存/文件、作业和处理器以及 I/O 设备）地管理。

### 分时系统

分时系统地特征除了继承自多道批处理系统系统的多路性外，还有独立性、及时性和交互性。

实时系统处理的任务现在可以是由和用户交互来实时获得并执行。多个用户可以同时通过不同的中断来访问同一台机器，并由操作系统来分配资源处理用户的请求。

操作系统相比起批处理多道程序设计，其主要区别在于指令来源由作业控制脚本给出转为用户由终端实时给出，以及优化目标从处理器使用最大化（无交互，总执行时间最小化）到相应时间最小化（实时等待时间最小化）。

### 实时系统

相比器分时系统，实时系统多了实时性和可靠性：系统需要在限定的时间内正确完成任务。

根据限定时间的强制性，可以分为软实时和硬实时系统。软实时系统要求各个任务越快越好，而硬实时则要求必须在规定时间内完成且执行无误。大多数实时系统是两者的结合。

## 操作系统的主要功能

可以分为对于进程管理、调度和资源的管理、内存管理及信息保护和安全。

### 进程

进程是一个正在执行的程序，是计算机中正在运行的程序的一个实例，是可分配给处理器并由处理器执行的一个实体。

进程和程序、作业的不同在于进程是一个动态的概念，只存在于内存中且具有生命周期，一旦完成或者被杀死就会被清出内存。而程序则是静态的，可以在内存也可以在外存中，一个进程也需要包含可执行的二进制程序。通过多次执行，一个程序可以对应多个进程。通过调用关系，一个进程也可能对应多个程序。作业则是一次进程的执行任务，一个被载入内存执行的作业叫做进程。（我瞎 yy 的）

进程由三部分组成：

- 一段可执行程序
- 程序所需要的相关数据（如内存空间中的变量以及缓冲区等）
- 程序的上下文，也称为进程状态，是操作系统用来管理和控制进程所需的所有数据

由此，一个进程在内存中一般也分为三个区域：程序区、数据区和上下文。

### 存储管理

对于内存而言，存储管理需要做到：

- 进程隔离和保护：每个进程拥有各自的资源，互不干扰，应用程序间的数据不能相互访问
- 自动分配和管理：动态分配各种资源，如堆栈空间
- 模块化程序设计：在程序执行过程中动态链接和释放所需的程序库

而对于磁盘和文件系统而言，则需要做到有序管理、快速存储和永久存储。（我瞎 yy 的+1

而实现这些目标则需要使用文件系统和虚拟存储，

### 信息保护和安全

操作系统的信息安全体现在以下四个方面：

- 可用性：保护系统不被中断运行
- 保密性：保证用户不能读取未授权访问的数据
- 数据完整性：保护数据不被未授权修改
- 可靠性：涉及用户身份的正确认证和消息或数据的合法性

> 信息保护和安全不是考的重点

### 调度和资源管理

对于进程竞争使用的各种资源的分配。

需要考虑的因素有

- 公平性：所有参与竞争某一特定资源的进程都能几乎相等且公平地访问资源
- 有差别的相应性：区分进程类型且可动态调整
- 有效性：最大化吞吐量和最小化响应时间，需要找到平衡点以折中处理矛盾需求

## 操作系统的目标与功能

四个管理：

- 处理机管理：通俗点就是 CPU 的调度和进程的调度同步通信
- 储存器管理：内存的分配管理和保护共享
- 设备的管理：管理 I/O 设备，为程序提供稳定易用的接口
- 文件管理：字面意思

## 操作系统的基本特征

> 似乎是重点

- 并发性（最重要，最基本的特性，其他特征的前提）
- 共享性（最基本的特征）：完成对于资源的共享和访问冲突控制
- 虚拟性：将单个物理实体抽象为逻辑实体
  - 时分复用：虚拟处理器，虚拟设备（打印机 SPOOLING 技术）
  - 空分复用：虚拟磁盘和内存
- 异步性：程序的执行具有不确定性：
  - 何时执行
  - 执行顺序
  - 完成运行所需时间

## 操作系统的体系结构

编写操作系统是一个大工程，因此需要良好的架构。操作系统的体系结构经历了四代变革

- 无结构：字面意义，特点是难以调试和维护
- 模块化结构：将功能分为若干个模块，各个模块间通过接口
- 分层式结构：类似于网络分层协议，各层利用低层提供的接口实现某些功能，再通过接口向其高层提供服务。在这一结构下进程也根据运行所在的层次分为用户模式和特权模式及其他中间态
- 微内核：多个模块实现各自的功能，扁平式管理
