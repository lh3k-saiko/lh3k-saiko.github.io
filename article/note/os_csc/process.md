# 进程控制

## 什么是进程

并发性是现代操作系统最重要的特征之一，进程是操作系统最重要的抽象概念之一。

并发基于进程，即进程是系统用于并发的载体。

### 顺序执行和并发执行

并发是指两个或多个时间在同一时间间隔内发生。而并行则是在同一时刻发生。在单核系统上我们只探究并发。

顺序执行是严格按照顺序先后执行，具有 **顺序性、封闭性和可再现性**。

而并发执行只有一个约束性的有向无环图，意味着一些进程和程序段的执行顺序是可以由操作系统决定的。并发执行具有 **间断性、失去封闭性和不可再现性**。

### 进程概念

为了控制多道程序能够正确地并发执行，引入进程。

进程由程序代码、程序代码相关数据及对应的进程控制块组成。

进程有四个特征：

- 动态性（最基本的特征）：进程是一个动态的概念，存在生命周期。
- 并发性（重要特征）：多个进程能同时存在于内存（包括虚拟内存）中同时运行。
- 独立性：除非采用通信手段，否则各个进程的运行互不干扰
- 异步性：进程间的运行互不干扰

引入进城后带来了三个挑战：空间开销、时间开销和控制复杂度。

## 进程状态

现代操作系统大多至少拥有以下七种进程状态：

- 新建：系统已经为进程在内存中创建了 PCB，但是还没有为进程分配资源
- 就绪：进程在内存中，且已经具备所有可以运行的条件
- 执行：进程在内存中，且正在处理器上执行
- 阻塞：进程在内存中，但是目前不能被调度执行，这是往往是由于被主动阻塞或者等待某些事件发生，如 I/O 请求
- 挂起/就绪：和就绪类似，但是进程在外存中，需要换入后才能执行
- 挂起/就绪：和阻塞类似，但是进程在外存中，在等待的事件发生后进入“挂起/就绪”

## 进程描述

### PCB

PCB 是进程存在的唯一标识，常驻于内存中，不会因进程挂起而换出。

PCB 中的信息分为以下三类：

- 进程表示符：包括内部标识符（pid），父进程表示符和用户表示符
- 处理机状态信息：包括程序运行时的寄存器和栈指针等，主要用于保存现场，便于之后重新调度时恢复
- 进程控制信息
  - 进程状态
  - 进程调度需要用的一些信息，如优先级、已等待的 CPU 时间之和、已执行时间等
  - 进程进入阻塞态的原因
  - 链接指针（用于维护维护阻塞/就绪队列）
  - 进程间通信量
  - 数据地址（段表和页表地址）
  - 资源所有权及其使用情况（如打开/写入文件等）

多个进程块之间的组织方式有两种，一种是为所有进程创建多个索引表。另一种则是利用链表将 PCB 块串联起来。而后者根据串联规则，又可分为单一队列（所有 PCB 块拼成一个队列）和多级队列（相同状态的 PCB 块连接成一个队列）。

## 进程模式

大多数硬件和操作系统都至少支持两种进程模式：内核模式和用户模式。

一些 CPU 指令只能在内核模式下运行。且用户模式下只能访问自己的虚拟内存，而内核模式可以直接访问所有物理内存。

处理器通过程序状态字寄存器（PSWR）来记录当前处理器处在什么模式下。

通过不同模式的区分和模式切换，系统可以将一些危险的操作封装为一些系统调用，从而避免用户直接接触一些危险操作同时提供服务。

### 内核

内核是操作系统中包含重要系统功能的部分，不会被换出内存。

内核的功能有：

- 资源管理
  - 进程管理：包括上文中的进程的创建、调度、切换、分派、同步、通信以及终止均需要由内核完成
  - 存储管理：为进程提供段页式空间管理
  - I/O 设备管理：为 I/O 设备创建缓冲区，并为程序分配 I/O 通道和设备等
- 支撑功能：
  - 中断处理：总而言之非常重要
  - 时钟管理
  - 记账（统计、监测）功能

### 模式切换

模式切换是由系统调用或中断产生的。出现中断时，系统首先会记录当前进程的 PC，然后将 PC 值为中断处理程序的开始地址，并以内核模式执行中断程序。完成中断程序后再返回用户模式并还原现场。

模式切换不一定导致进程切换（如 I/O 操作只需要将模式“升级”为内核来完成，不需要切换）。但是用户模式进行进程切换则必须要先切换为内核模式才能完成。

## 线程

早期的进程是资源分配和调度的基本单元，但是这样进程间通信就不是很方便。

为此引入了线程（轻量级进程），线程是调度的基本单元，进程是资源分配的基本单元。同一进程下的线程共享该进程资源。

线程相较于进程拥有更小的系统开销，且进程内线程的通信和同步更加容易。

线程由三种类型：用户级线程、内核级线程及混合线程。
